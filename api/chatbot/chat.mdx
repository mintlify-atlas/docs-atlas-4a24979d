---
title: 'Send Chat Message'
api: 'POST /chatbot/message'
---

Sends a message to the SaludMap AI chatbot powered by Google Gemini and receives a response. The chatbot can answer questions about SaludMap features, healthcare information, and help users navigate the platform.

## Request Body

<ParamField body="message" type="string | string[]" required>
  The message to send to the chatbot. Can be a single string or an array of strings (which will be joined with spaces). Must be between 1-1000 characters.
</ParamField>

<ParamField body="userId" type="string">
  Optional user identifier for tracking conversation history
</ParamField>

<ParamField body="sessionId" type="string">
  Optional session identifier for maintaining conversation context
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the request was successful
</ResponseField>

<ResponseField name="data" type="object">
  Response data object
  
  <ResponseField name="userMessage" type="string">
    The processed message sent by the user
  </ResponseField>
  
  <ResponseField name="botResponse" type="string">
    The AI-generated response from the chatbot
  </ResponseField>
  
  <ResponseField name="timestamp" type="string">
    ISO 8601 timestamp of when the response was generated
  </ResponseField>
</ResponseField>

## Example Requests

### Simple Message

```json
{
  "message": "¿Cómo puedo reservar un turno médico?"
}
```

### Message with Context

```json
{
  "message": "¿Qué especialidades están disponibles?",
  "userId": "user-123",
  "sessionId": "session-abc-456"
}
```

### Array Message (Multiple Parts)

```json
{
  "message": [
    "Necesito un turno con un cardiólogo",
    "en el Hospital Central",
    "para la próxima semana"
  ]
}
```

## Error Responses

### Empty Message

```json
{
  "statusCode": 400,
  "message": "El mensaje no puede estar vacío"
}
```

### Validation Error (Message Too Long)

```json
{
  "statusCode": 400,
  "message": "Error de validación",
  "details": [
    {
      "code": "too_big",
      "maximum": 1000,
      "type": "string",
      "inclusive": true,
      "message": "El mensaje es demasiado largo",
      "path": ["message"]
    }
  ]
}
```

### Service Error

```json
{
  "statusCode": 500,
  "message": "Error al procesar el mensaje"
}
```

```bash

}
```

## Validation Rules

- Message must not be empty after trimming whitespace
- Message length must be between 1 and 1000 characters
- If message is an array, it will be joined with spaces
- userId and sessionId are optional but recommended for better conversation tracking
