description: 'Retrieve detailed information about a specific establishment by its unique identifier'
---

## Overview

Fetch complete details for a healthcare establishment using its database ID. Returns full information including reviews with user details and available specialties.

## Path Parameters

<ParamField path="id" type="number" required>
  Unique identifier of the establishment.
  
  **Example:** `42`
</ParamField>

## Error Responses

### 400 Bad Request

Returned when the ID parameter is not a valid integer:

```json
{
  "statusCode": 400,
  "message": "Validation failed (numeric string is expected)",
  "error": "Bad Request"
}
```

### 404 Not Found

Returned when no establishment exists with the specified ID:

```json
{
  "statusCode": 404,
  "message": "Establecimiento no encontrado",
  "error": "Not Found"
}
```

## Use Cases

- **Detail Page**: Display complete establishment information on a dedicated page
- **Review System**: Show all reviews for an establishment
- **Specialty Lookup**: Find what medical services are available
- **Direct Access**: Navigate directly to a specific establishment via URL
- **Data Relationships**: Access full relational data (reviews, specialties, users)

## Related Endpoints

- `GET /establecimientos/:id/resenias` - Get reviews with statistics
- `GET /establecimientos/:id/especialidades` - Get only specialties
- `GET /establecimientos/coords/:lat/:lng` - Find by coordinates instead of ID

## Notes

- Reviews are ordered by creation date (most recent first)
- User information in reviews is limited to id, nombre, and apellido for privacy
- Specialties include relationship data (horariosDisponibles from the join table)
- Returns 404 if the establishment doesn't exist (unlike coords endpoint which returns 200 with a message)
- The `metadata` field can contain any JSON structure
- Related entities (reviews, specialties) are eagerly loaded in a single query
