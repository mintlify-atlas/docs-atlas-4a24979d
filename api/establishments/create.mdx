## Overview

Create a new healthcare establishment record. This endpoint validates that no establishment exists at the exact same coordinates before creating.

## Request Body

<ParamField body="lat" type="number" required>
  Latitude coordinate of the establishment.
  
  **Example:** `-34.6037`
</ParamField>

<ParamField body="lng" type="number" required>
  Longitude coordinate of the establishment.
  
  **Example:** `-58.3816`
</ParamField>

<ParamField body="nombre" type="string" required>
  Name of the establishment.
  
  **Example:** `"Hospital General de Agudos"`
</ParamField>

<ParamField body="tipo" type="string" required>
  Type of establishment.
  
  **Common values:** `hospital`, `clinic`, `doctors`, `veterinary`
  
  **Example:** `"hospital"`
</ParamField>

<ParamField body="direccion" type="string">
  Physical address of the establishment.
  
  **Example:** `"Av. Corrientes 1234, Buenos Aires"`
</ParamField>

<ParamField body="telefono" type="string">
  Contact phone number.
  
  **Example:** `"+54 11 4567-8900"`
</ParamField>

<ParamField body="horarios" type="string">
  Operating hours information.
  
  **Example:** `"Lunes a Viernes 8:00-20:00, Sábados 9:00-13:00"`
</ParamField>

<ParamField body="metadata" type="object">
  Additional metadata as a JSON object.
  
  **Example:** `{"emergency": true, "beds": 150, "parking": true}`
</ParamField>

## Response

Returns the created establishment object.

<ResponseField name="id" type="number">
  Unique identifier assigned to the new establishment
</ResponseField>

<ResponseField name="nombre" type="string">
  Establishment name
</ResponseField>

<ResponseField name="tipo" type="string">
  Establishment type
</ResponseField>

<ResponseField name="lat" type="number">
  Latitude coordinate
</ResponseField>

<ResponseField name="lng" type="number">
  Longitude coordinate
</ResponseField>

<ResponseField name="direccion" type="string | null">
  Street address
</ResponseField>

<ResponseField name="telefono" type="string | null">
  Contact phone number
</ResponseField>

<ResponseField name="horarios" type="string | null">
  Operating hours
</ResponseField>

<ResponseField name="metadata" type="object">
  Additional metadata (defaults to `{}` if not provided)
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp of creation
</ResponseField>

<ResponseField name="updatedAt" type="string">
  ISO 8601 timestamp of last update
</ResponseField>

## Example Request

```bash
curl -X POST "https://api.saludmap.com/establecimientos" \
  -H "Content-Type: application/json" \
  -d '{
    "lat": -34.6037,
    "lng": -58.3816,
    "nombre": "Hospital General de Agudos",
    "tipo": "hospital",
    "direccion": "Av. Corrientes 1234, Buenos Aires",
    "telefono": "+54 11 4567-8900",
    "horarios": "24 horas",
    "metadata": {
      "emergency": true,
      "beds": 200,
      "parking": true,
      "wheelchair_accessible": true
    }
  }'
```

## Error Responses

### 400 Bad Request

Returned when required fields are missing or invalid:

```json
{
  "statusCode": 400,
  "message": [
    "lat must be a number",
    "lng must be a number",
    "nombre should not be empty",
    "tipo should not be empty"
  ],
  "error": "Bad Request"
}
```

### 409 Conflict

Returned when an establishment already exists at the specified coordinates:

```json
{
  "statusCode": 409,
  "message": "Ya existe un establecimiento en esas coordenadas",
  "error": "Conflict"
}
```

## Validation Rules

- `lat` and `lng` must be valid numbers
- `nombre` and `tipo` cannot be empty strings
- Coordinates must be unique (no duplicate establishments at same location)
- Optional fields (`direccion`, `telefono`, `horarios`) can be omitted or null
- `metadata` defaults to empty object `{}` if not provided

## Notes

- The endpoint checks for existing establishments at exact coordinates before creation
- Coordinates are compared with exact equality (consider using `/establecimientos/find-or-create` for upsert behavior)
- The `metadata` field accepts any valid JSON structure for flexibility
- Timestamps are automatically generated using the database server time
