---
title: 'Update Appointment'
api: 'PUT /turnos/:id'
---

Updates an existing appointment. This endpoint is primarily used to cancel appointments or modify their details.

## Path Parameters

<ParamField path="id" type="number" required>
  The unique identifier of the appointment to update
</ParamField>

## Request Body

<ParamField body="action" type="string">
  The action to perform on the appointment. Use "cancelar" to cancel the appointment.
</ParamField>

<ParamField body="fecha" type="string">
  New date for the appointment in ISO 8601 format (e.g., "2026-03-20")
</ParamField>

<ParamField body="hora" type="string">
  New time for the appointment in HH:MM format (e.g., "14:30")
</ParamField>

## Response

<ResponseField name="id" type="number">
  The unique identifier of the updated appointment
</ResponseField>

<ResponseField name="usuarioId" type="number">
  The ID of the user who booked the appointment
</ResponseField>

<ResponseField name="establecimientoId" type="number">
  The ID of the healthcare establishment
</ResponseField>

<ResponseField name="especialidadId" type="number">
  The ID of the medical specialty
</ResponseField>

<ResponseField name="fecha" type="string">
  The date of the appointment
</ResponseField>

<ResponseField name="hora" type="string">
  The time of the appointment
</ResponseField>

<ResponseField name="estado" type="string">
  The updated status of the appointment ("pendiente", "completado", "cancelado")
</ResponseField>

<ResponseField name="updatedAt" type="string">
  Timestamp when the appointment was last updated
</ResponseField>

## Example Requests

### Cancel an Appointment

```json
{
  "action": "cancelar"
}
```

### Reschedule an Appointment

```json
{
  "fecha": "2026-03-25",
  "hora": "16:00"
}
```

### Full Update Example

```bash
PUT /turnos/1001
Content-Type: application/json

{
  "action": "cancelar"
}
```
