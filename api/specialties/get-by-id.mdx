---
title: 'Get Specialty by ID'
api: 'GET /especialidades/:id'
---

Retrieves a single medical specialty by its unique identifier. This is a public endpoint that does not require authentication.

## Path Parameters

<ParamField path="id" type="number" required>
  The unique identifier of the specialty to retrieve
</ParamField>

## Response

<ResponseField name="id" type="number">
  The unique identifier for the specialty
</ResponseField>

<ResponseField name="nombre" type="string">
  The name of the medical specialty
</ResponseField>

<ResponseField name="descripcion" type="string">
  A description of the specialty (optional)
</ResponseField>

<ResponseField name="createdAt" type="string">
  Timestamp when the specialty was created
</ResponseField>

## Example Request

```bash
curl -X GET https://api.example.com/especialidades/1
```

## Example Response

```json
{
  "id": 1,
  "nombre": "Cardiología",
  "descripcion": "Especialidad médica que se ocupa de las enfermedades del corazón y del aparato circulatorio",
  "createdAt": "2026-01-15T10:00:00.000Z"
}
```

## Error Responses

<ResponseField name="404 Not Found">
  Returned when the specialty with the specified ID does not exist
  
  ```json
  {
    "statusCode": 404,
    "message": "Especialidad con ID 999 no encontrada",
    "error": "Not Found"
  }
  ```
</ResponseField>

## Use Cases

- Display detailed information about a specific specialty
- Verify specialty existence before assignment
- Fetch specialty details for display in UI components
