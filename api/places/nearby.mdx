## Overview

This endpoint performs a proximity-based search for healthcare establishments using OpenStreetMap data. It's designed for finding the nearest hospitals, clinics, doctors offices, and other medical facilities within a specified radius.

## Use Cases

- **Emergency Services**: Find the nearest hospital or clinic
- **Location-Based Search**: Show all medical facilities on a map
- **Filter by Type**: Search only for specific facility types
- **Radius Control**: Expand or narrow search area based on user needs

## Query Parameters

<ParamField query="lat" type="number" required>
  Center point latitude for the proximity search.
  
  **Example:** `-34.6037` (Buenos Aires)
</ParamField>

<ParamField query="lng" type="number" required>
  Center point longitude for the proximity search.
  
  **Example:** `-58.3816` (Buenos Aires)
</ParamField>

<ParamField query="types" type="string">
  Amenity types to include in the search. Can be:
  - Single value: `types=hospital`
  - Multiple comma-separated: `types=hospital,clinic,doctors`
  - Array format: `types[]=hospital&types[]=clinic`
  
  **Available types:**
  - `hospital` - General hospitals
  - `clinic` - Medical clinics
  - `doctors` - Doctor's offices
  - `veterinary` - Veterinary clinics
  - `pharmacy` - Pharmacies (if supported)
  - Other OSM amenity types
  
  **Default:** `hospital,clinic,doctors,veterinary`
</ParamField>

<ParamField query="radius" type="number">
  Search radius in meters.
  
  **Default:** `3000` (3km)
  
  **Recommended ranges:**
  - Urban areas: 1000-5000m
  - Suburban: 5000-10000m
  - Rural: 10000-20000m
</ParamField>

## Response Structure

<ResponseField name="lugares" type="array">
  Array of nearby places from OpenStreetMap.
  
  <ResponseField name="id" type="number">
    OpenStreetMap element ID
  </ResponseField>
  
  <ResponseField name="type" type="string">
    OSM element type: `node`, `way`, or `relation`
  </ResponseField>
  
  <ResponseField name="lat" type="number">
    Latitude of the facility
  </ResponseField>
  
  <ResponseField name="lon" type="number">
    Longitude of the facility
  </ResponseField>
  
  <ResponseField name="tags" type="object">
    Metadata tags from OpenStreetMap:
    
    <ResponseField name="name" type="string">
      Facility name
    </ResponseField>
    
    <ResponseField name="amenity" type="string">
      Type of amenity (hospital, clinic, etc.)
    </ResponseField>
    
    <ResponseField name="addr:*" type="string">
      Address components (street, housenumber, city, postcode)
    </ResponseField>
    
    <ResponseField name="phone" type="string">
      Contact phone number
    </ResponseField>
    
    <ResponseField name="opening_hours" type="string">
      Operating hours in OSM format
    </ResponseField>
    
    <ResponseField name="emergency" type="string">
      Emergency service availability (yes/no)
    </ResponseField>
    
    <ResponseField name="beds" type="string">
      Number of beds (for hospitals)
    </ResponseField>
    
    <ResponseField name="website" type="string">
      Official website URL
    </ResponseField>
  </ResponseField>
</ResponseField>

<ResponseField name="lejania" type="number">
  The radius used for the search (in meters)
</ResponseField>

## Examples

### Basic Nearby Search

```bash
curl "https://api.saludmap.com/places?lat=-34.6037&lng=-58.3816"
```

### Search Only Hospitals Within 10km

```bash
curl "https://api.saludmap.com/places?lat=-34.6037&lng=-58.3816&types=hospital&radius=10000"
```

### Multiple Facility Types

```bash
curl "https://api.saludmap.com/places?lat=-34.6037&lng=-58.3816&types=hospital,clinic,doctors&radius=5000"
```

## Implementation Notes

- Uses Overpass API with automatic failover between multiple mirrors
- 20-second timeout per mirror attempt
- Queries nodes, ways, and relations for comprehensive coverage
- Results ordered by distance from center point (handled by Overpass)
- No pagination - returns all results within radius
- Consider caching results for frequently searched locations
