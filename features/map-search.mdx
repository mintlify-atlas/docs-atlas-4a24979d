## Overview

SaludMap's map-based search is the core feature that allows users to discover healthcare facilities in their area. Built with Leaflet and OpenStreetMap, it provides real-time geolocation, facility filtering, and offline support.

## Location Services

### GPS Tracking

The map automatically tracks your location using the browser's Geolocation API:

```jsx
// Auto-tracking starts on component mount
locationService.startWatching();

// Location updates trigger map recentering
const handleLocationChange = async (location) => {
  setCurrentLocation(location);
  mapRef.current.setView([location.lat, location.lng], 15);
  await fetchNearbyPlaces(location);
};
```

<Info>
Location accuracy is displayed in the bottom-left corner, showing precision in meters.
</Info>

### Location Sources

SaludMap tracks three types of location sources:

- **GPS**: Continuous tracking from device location services
- **Calibrated**: One-time high-accuracy position update
- **Manual**: User-dragged marker position

```jsx
// Calibrate for high-accuracy position
const handleCalibrate = async () => {
  const location = await locationService.calibratePosition();
  mapRef.current.setView([location.lat, location.lng], 15);
};

// Manual positioning via draggable marker
const handleMarkerDrag = async (event) => {
  const { lat, lng } = event.target.getLatLng();
  await locationService.setManualLocation(lat, lng);
};
```

## Facility Discovery

### Nearby Places API

The map queries healthcare facilities based on your current location:

```jsx
const fetchNearbyPlaces = async (location) => {
  const selectedTypes = Object.keys(filters).filter(k => filters[k]);
  const types = selectedTypes.join(',') || 'hospital,clinic,doctors,veterinary';
  
  const response = await axios.get(
    `/api/places?lat=${location.lat}&lng=${location.lng}&types=${types}`
  );
  
  setLugares(normalizeApiResponse(response.data));
};
```

## Filtering

### Filter Controls

Users can filter visible facilities by type:

```jsx
const [filters, setFilters] = useState({
  hospital: true,
  clinic: true,
  doctors: true,
  veterinary: true
});

const visibleLugares = lugares.filter(l => {
  const tipo = l.type || l.tipo || 'default';
  return !!filters[tipo];
});
```

## Saved Locations

### Saving Locations

Users can save custom locations for quick access:

```jsx
const handleSaveLocation = async (locationData) => {
  await saveNamedLocation(
    locationData.name,
    locationData.lat,
    locationData.lng,
    locationData.description
  );
};
```

<Note>
Saved locations are stored in IndexedDB and persist across sessions.
</Note>

### Location Management

- **Save Current Location**: Opens modal to name and save current map position
- **View Saved Locations**: Browse and jump to saved locations
- **Delete Locations**: Remove saved locations from storage

## Map Controls

### Navigation Bar Integration

The map responds to actions from the global navigation bar:

```jsx
useEffect(() => {
  const listener = (e) => {
    switch(e.detail) {
      case 'calibrate': handleCalibrate(); break;
      case 'return-gps': handleReturnToGPS(); break;
      case 'download-offline': handleDownloadOffline(); break;
      case 'save-location': setShowSaveLocationModal(true); break;
      case 'view-locations': setShowSavedLocationsList(true); break;
      case 'filters': setShowFiltersModal(true); break;
    }
  };
  window.addEventListener('map-action', listener);
}, []);
```

### User Interaction Detection

The map detects when users manually pan or zoom to prevent automatic recentering:

```jsx
const onUserInteractionStart = () => {
  userInteracting.current = true;
};

map.on('movestart', onUserInteractionStart);
map.on('zoomstart', onUserInteractionStart);
```

## Facility Details

### Establishment Info Panel

Clicking a facility marker opens a detailed panel:

```jsx
const handlePlaceSelect = async (lugar) => {
  setSelectedPlace(lugar);
  const est = await establecimientosService.findOrCreate(lugar);
  setSelectedEstablecimiento(est);
  if (onEstablishmentSelect) onEstablishmentSelect(est);
};
```

The panel displays:
- Facility name and type
- Address and contact information
- Opening hours (if available)
- Review statistics
- Action buttons (appointments, reviews)

## Performance Optimizations

<AccordionGroup>
  <Accordion title="Icon Caching">
    Marker icons are cached to avoid recreating them on every render:
    ```jsx
    const iconCache = {};
    const getIconForType = (type) => {
      if (!iconCache[type]) {
        iconCache[type] = createDivIcon(type);
      }
      return iconCache[type];
    };
    ```
  </Accordion>
  
  <Accordion title="Debounced Updates">
    Location updates are debounced to prevent excessive API calls and map recentering.
  </Accordion>
  
  <Accordion title="Lazy Loading">
    Facility details are loaded on-demand when markers are clicked, not when initially displayed.
  </Accordion>
</AccordionGroup>

<Tip>
For best performance, limit the number of visible markers by using filters or reducing the search radius.
</Tip>
