## Overview

SaludMap offers three comprehensive health insurance plans that users can contract directly through the platform. The system includes plan selection, secure checkout, PDF contract generation, and email confirmation.

<Warning>
  **Authentication Required**: Users must be logged in to contract insurance plans. Unauthenticated users will be prompted to log in or register.
</Warning>

## PDF Generation

### Insurance Card PDF

The system generates a professional insurance card PDF using **jsPDF**:

```typescript PDF Generator (PDFGenerator.ts:20-52)
export class PDFGenerator {
  static generateInsuranceCard(plan: InsurancePlan, userInfo: UserInfo): void {
    const doc = new jsPDF();

    // Header
    doc.setFontSize(20);
    doc.text('Tarjeta de Seguro Médico', 20, 30);

    // Plan information
    doc.setFontSize(16);
    doc.text(`Plan: ${plan.name}`, 20, 50);
    doc.setFontSize(12);
    doc.text(`Precio: $${plan.price}/mes`, 20, 65);

    // User information
    doc.setFontSize(14);
    doc.text('Información del Asegurado:', 20, 85);
    doc.setFontSize(12);
    doc.text(`Nombre: ${userInfo.name}`, 20, 100);
    doc.text(`Email: ${userInfo.email}`, 20, 115);
    doc.text(`Teléfono: ${userInfo.phone}`, 20, 130);
    doc.text(`Dirección: ${userInfo.address}`, 20, 145);

    // Coverage
    doc.setFontSize(14);
    doc.text('Cobertura:', 20, 165);
    doc.setFontSize(12);
    plan.coverage.forEach((item, index) => {
      doc.text(`• ${item}`, 25, 180 + (index * 15));
    });

    // Save the PDF
    doc.save(`seguro-${plan.name.toLowerCase().replace(/\s+/g, '-')}.pdf`);
  }
}
```

## Email Confirmation

### Confirmation Email Service

After successful checkout, the system sends a confirmation email using EmailJS:

```tsx Email Processing (CheckoutModal.tsx:72-109)
const handleSubmit = async (e: React.FormEvent) => {
  e.preventDefault();
  setIsProcessing(true);

  try {
    initializeEmailJS();

    // Generate and download PDF
    const pdfDownloaded = downloadInsurancePDF(plan, userInfo);
    if (!pdfDownloaded) {
      throw new Error('Error generando el PDF de la tarjeta de seguro');
    }

    // Send confirmation email
    const emailResult = await sendInsuranceConfirmationEmail(plan, userInfo);

    if (emailResult.success) {
      setEmailSent(true);
      toast.success(
        `¡Seguro contratado! Confirmación enviada a ${userInfo.email}. Póliza: ${emailResult.orderId}`
      );
    } else {
      toast.info('¡Seguro contratado! Se descargó tu tarjeta. El correo de confirmación no pudo enviarse, pero tu seguro está activo.');
    }

    onClose();
  } catch (error) {
    toast.error(`Error al procesar el seguro: ${error.message}`);
  } finally {
    setIsProcessing(false);
  }
};
```

<Info>
  The system provides graceful degradation: if email delivery fails, the PDF still downloads and the insurance is considered active. Users receive a notification about the email failure.
</Info>

## Error Handling

The system includes comprehensive error handling:

<Steps>
  <Step title="PDF Generation Failure">
    Throws error and displays toast notification
  </Step>
  <Step title="Email Delivery Failure">
    Insurance is still considered active, user receives info notification
  </Step>
  <Step title="Form Validation">
    HTML5 validation prevents submission with missing fields
  </Step>
  <Step title="Authentication Required">
    Redirects to login modal if user is not authenticated
  </Step>
</Steps>

<Tip>
  The insurance system gracefully handles partial failures. Even if the email confirmation fails, users still receive their insurance card PDF and the plan is activated.
</Tip>
